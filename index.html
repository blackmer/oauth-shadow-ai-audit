OAuth & Shadow AI Control Audit Dashboard

v0.1.0

Import Data Files

This dashboard audits OAuth application access in your Google Workspace and Microsoft 365 tenants. All processing happens in your browser — no data is transmitted anywhere.

Export the files below from your admin consoles, then upload them here. You can load files from one or both platforms. The dashboard degrades gracefully when files are missing — see the feature matrix below each section.

Heads up: Admin console UIs change. If menu labels in the steps below don't match exactly what you see, follow the equivalent path — the file schemas the dashboard expects are the source of truth.

Google Workspace

Two files from the Google Admin Console. Requires Super Admin access.

Export instructions

G1 — Accessed Apps (required for Google audit)

  1. Sign in to admin.google.com as a Super Admin.
  2. Navigate to Security → Access and data control → API controls → App access control.
  3. Click MANAGE APP ACCESS.
  4. In the tile carousel, find the Accessed apps tile (scroll right if needed).
  5. Click View list.
  6. Click Download list at the top of the list.
  7. In the format dialog, select Comma-separated values (.csv) and click Download.
  8. The file appears in your Tasks list when ready — open from there.

Provides: app inventory, scopes by service, verification status, ownership classification.

G2 — OAuth Audit Log (recommended)

  1. In the Admin Console, navigate to Reporting → Audit and investigation → OAuth log events.
  2. Set date range as wide as available (Google retains ~6 months).
  3. Click Download. Save the file.

Provides: grant/revoke timeline, grantor identity, last consent event dates.

Microsoft Entra ID / Microsoft 365

One CSV from the Entra admin center plus up to three JSON files from Microsoft Graph Explorer. Requires at minimum Global Reader or Application Administrator role.

Export instructions

M1 — Enterprise Applications CSV

  1. Sign in to entra.microsoft.com.
  2. Browse to Entra ID → Enterprise apps → All applications.
  3. Click Download (Export).
  4. In the Bulk Jobs Default panel, optionally rename the file, then click Start bulk operation.
  5. Click the link below the button to view bulk operation status.
  6. When the operation completes, click the file link on the Bulk Operations page to download.

Provides: app inventory metadata (names, IDs, creation dates). Does not include scopes or permissions.

M2, M3, M4 — Graph Explorer JSON files

  1. Open developer.microsoft.com/en-us/graph/graph-explorer and sign in with your tenant admin account.
  2. Consent to the required permissions when prompted (see table below).
  3. Run each query, then copy the full JSON response and save it as a .json file.
